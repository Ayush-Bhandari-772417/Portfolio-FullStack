    - Basic structure
        - \.next
            - Automatically generated by Next.js
            - Stores: build output, cached server renders and optimized assets
            - Deleted safely when rebuilding (npm run build recreates it)
            - caches all the information of the current structure to run the app
        - \node_modules
            - Contains all installed dependencies
            - Generated from package.json
            - Never edit manually
            - Removed & recreated using npm install
            - contains all the packages required to run the app
        - \public
            - photos, pdfs, etc. required for the project
            - Stores static assets like images and PDFs
            - Files here are served directly at /filename
        - \src\app\layout.tsx
            - determines the layout of UI for the whole project
            - the main page gets rendered within the layout connected by children
            - Global layout wrapper for the entire app
            - It contains <html>, <body>, Navbar & Footer
            - Every page renders inside children
            - Loaded once to improve the performance
        - \src\app\page.tsx
            - default page which is loaded on loading the base URL
            - Server component
            - Fetches settings and profile data
            - renders different sections on portfolio page
            - generates the metadata and OG tag for the home/portfolio page
        - \src\components\commonSections\Footer.tsx
            - design of footer for every pages of the project
            - Static content + links
            - Improves consistency
        - \src\components\commonSections\Navbar.tsx
            - design of navbar for every pages of the project
            - Navigation bar for all pages
            - Used inside layout.tsx
            - Client component (interactivity)
        - \src\lib\data.ts
            - a single file for fetching all the data from backend
            - used by every component by importing the related functions
            - Centralized data-fetching layer
            - It contains all API calls along with caching & revalidation logic
            - Used by server components
            - Prevents duplicated fetch logic
        - \src\styles\globals.css
            - css for the whole project
            - import the css and tailwind, scroll and all
        - \src\types\index.ts
            - defines all the models so that data shouldn't be defined again and again
            - Ensures type safety, autocomplete and fewer runtime bugs
        - \.env.local
            - Environment variables
            - Not committed to git
            - Stores secrets safely like recaptcha key, revalidation key and base url
        - \.gitignore
            - the files and folders that should not be uploaded along with other files of the project
            - Tells Git which files/folders NOT to track
        - \eslint.config.mjs
            - Configuration file for ESLint (code quality + linting)
            - It controls code style rules & error warnings
            - It is React/Next.js best practices
        - \next-env.d.ts
            - Auto-generated TypeScript declaration file for Next.js
            - It enables
                - next/image
                - next/navigation
                - Next-specific types
        - \next.config.js
            - Configuration file for Next.js behavior
            - It is used for:
                - Image domains
                - Redirects / rewrites
                - Experimental features
                - Webpack tweaks
        - \package-lock.json
            - Exact dependency version lock file (npm)
        - \package.json
            - The heart of the project
            - It contains dependencies, Scripts (dev, build, start) and Project metadata
        - \postcss.config.mjs
            - It is the configuration for PostCSS.
            - It is used by tailwind CSS, autoprefixer and modern CSS transforms
        - \tsconfig.json
            - It contains typeScript compiler configuration
            - Used to control strictness, path aliases (@/components) and module resolution
